# Infobiz Intake — Чек-лист сбора данных

> Этот файл описывает, какие данные нужно собрать у клиента для сборки приложения по продаже инфопродуктов.
> Используется LLM-агентом при парсинге брифа и формой intake на фронтенде.

---

## Обязательные поля

### Бизнес
| Поле | Тип | Пример | Описание |
|------|-----|--------|----------|
| `name` | string | "Growth Academy" | Название школы / бренда |
| `slug` | string | "growth-academy" | URL-ключ |
| `locale` | "ru" \| "en" | "ru" | Язык интерфейса |
| `currency` | string | "RUB" | Валюта (Stars-оплата работает с любой) |

### Инфопродукты (минимум 1)
| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|----------|
| `name` | string | ✅ | Название продукта |
| `slug` | string | ✅ | URL-ключ (генерируется из name) |
| `type` | enum | ✅ | "article" \| "pdf" \| "course" \| "consultation" |
| `price` | number | ✅ | Цена (0 = бесплатный) |
| `description` | string | ❌ | Описание (2-3 предложения) |
| `content` | string | ❌ | Полный текст (для article) — markdown |
| `image` | file/url | ❌ | Обложка |
| `file_id` | string | ❌ | UUID файла в Directus (для pdf) |
| `external_url` | url | ❌ | Ссылка на курс/платформу |

**Типы продуктов:**
- `article` — текстовый контент, отображается прямо в приложении (поле `content`)
- `pdf` — скачиваемый файл (поле `file_id`)
- `course` — ссылка на внешнюю платформу (поле `external_url`)
- `consultation` — запись через lead-форму, цена за сессию

## Опциональные поля

### Автор / эксперт
| Поле | Тип | Описание |
|------|-----|----------|
| `authorName` | string | Имя эксперта |
| `authorTitle` | string | Должность / специализация |
| `authorBio` | string | О себе (2-3 предложения) |
| `authorPhoto` | file/url | Фото |
| `authorCredentials` | string[] | Регалии: ["10+ лет опыта", "500+ учеников"] |

### Брендинг
| Поле | Тип | По умолчанию | Описание |
|------|-----|:---:|----------|
| `primaryColor` | hex | #6366f1 | Основной цвет |
| `secondaryColor` | hex | #8b5cf6 | Акцент |
| `subtitle` | string | auto | Слоган |

### Маркетинг
| Поле | Тип | Описание |
|------|-----|----------|
| `stats` | array<{icon, value, label}> | Цифры соц. доказательства |
| `features` | array | 4 преимущества |
| `testimonials` | array | Отзывы учеников |
| `faq` | array<{question, answer}> | Часто задаваемые вопросы |

## Автозаполнение (LLM генерирует если не указано)

| Что | Откуда | Пример |
|-----|--------|--------|
| `slug` (бизнес) | из `name` | "Growth Academy" → "growth-academy" |
| `slug` (продукт) | из имени продукта | "Telegram Mini Apps Guide" → "tma-guide" |
| `subtitle` | из ниши | "Знания, которые окупаются" |
| `stats` | типовые | ["1,000+ учеников", "50 курсов", "4.9 рейтинг"] |
| `features` | типовые для infobiz | ["Доступ навсегда", "Поддержка автора", "Оплата Stars"] |
| `testimonials` | генерация | 3 отзыва учеников |
| `faq` | типовые | ["Бессрочный доступ?", "Возврат?", "На каком языке?", "Как получить?"] |
| `authorBio` | из имени + продуктов | Генерируется описание эксперта |
| `authorCredentials` | из кол-ва продуктов | ["N курсов", "Эксперт в <нише>"] |
| `description` | из названия + типа | Краткое описание продукта |

---

## Валидация на входе

- `name` не пустой
- Минимум 1 продукт с `name`, `type` и `price >= 0`
- `type` — одно из: article, pdf, course, consultation
- Для `article` — желательно `content` (иначе пустая страница)
- Для `pdf` — желательно `file_id` или URL файла для загрузки
- Для `course` — желательно `external_url`
- Для `consultation` — желательно `price > 0`
- Фото — jpg/png/webp, до 5MB
